پروژه میانی اول

با سلام و احترام

1-با فرمان mkdir یک دایرکتوری به نام git_learn ایجاد کردم.
سپس با فرمان git init دایرکتوری git_learn را به یک repository تبدیل کردم.

2- با فرمان git config نام کاربری و ایمیلم رو به صورت global کانفیگ کردم.

3-با استفاده از دستور touch یک فایل متنی به نام main.txt در rep ایجاد کردم.

4- با دستور git ststus وضعیت فایلهای rep رو چک کردم در اولین مرحله فقط یک فایل untracked در rep داشتم که هم.ن فایل main.txt بود

5- با فرمان git add فایل main.txt رو به مرحله staging بردم.

6- در فایل main.txt تغییراتی ایجاد کردم مثلا عبارت line1 رو تایپ کردم و بعد فایل رو با دستور git commit به کامیت تبدیل کردم و مجدد git status گرفتم که نشون میداد فایل main.txt کامیت شده است.
مجددا فایل main.txt رو تغییر دادم و git status گرفتم که نشون میداد فایل modify شده است. با دستور git add فایل رو مجددا به staging بردم و بعد با git commit آنرا کامیت کردم.

7-در این مرحله فایل جدیدی به rep اضافه کردم به نام first_git_txt فایل جدید رو به staging بردم بعد تغییراتی در آن ایجاد کردم و پیش ار آنکه فایل را مجددا به مرحله staging ببرم با فرمان git diff آخرین تغییرات ایجاد شده روی این فایل را مشاهده کردم.
سپس فایل را به staging بردم و بعد آنرا با کامیت کردم حالا git status گرفتم که نشان داد فایلی برای کامیت شدن وجود ندارد و working tree clean است.
برای مشاهده تغییرات در rep از فرمان git log استفاده کردم که تاریخچه rep و کلیه کامیتهای آنرا به من نشان داد. تا این مرحله هفت کامیت در rep وجود داشت.
برای مشاهده تغییرات یک فایل خاص مثلا فایل first_git_pr.txt از فرمان git blame first_git_pr.txt استفاده کردم و توانستم تغییرات خط به خط فایل رو بررسی کنم .
